## 1. 개요

Vibe Studio Fan 사전예약은 **플로팅 패널** 형태로 메인 페이지(`index.html`) 우측에 노출됩니다.

| 항목 | 내용 |
|------|------|
| 진입 | 우측 `PRE-REGISTER` 세로 버튼 클릭 |
| 가입 경로 | Google OAuth (추천) / 이메일 직접 입력 |
| 중복 신청 | 허용 (기존 항목 조정 가능) |
| 패널 위치 | 화면 우측 슬라이드인 (PC: 420px, 모바일: 전체화면) |

## 4. Step 1-A — Google 로그인 경로

### 4-1. 팝업 실행

`Google로 시작하기` 버튼 클릭 시:

1. 버튼 비활성화 + 텍스트: `Google 연결 중...`
2. 내부 상태: `google_pending`
3. 팝업 창 오픈: `window.open('/google_fan_oauth.php', 'vibeGoogleFanAuth', 'width=500,height=620')`
4. 팝업 상태 폴링 (800ms 간격) + 15분 타임아웃

### 4-2. 팝업 결과 처리 (`postMessage`)

| 상태값 | 의미 | 처리 |
|--------|------|------|
| `success` | 인증 성공, 이메일 수신 | Step 2로 이동 |
| `cancelled` | 사용자가 팝업 직접 닫음 | Step 0 유지, 안내 메시지 표시 |
| `no_email` | Google 계정에 이메일 없음 | 이메일 수동 입력 모드로 전환 |
| 기타 | 서버 오류 등 | Step 0으로 복귀 |

### 4-3. Step 2 진입 시 UI (Google 경로)

| 요소 | 상태 |
|------|------|
| 이메일 입력 필드 | **비활성화** (Google 이메일 자동 채워짐) |
| 인증 발송 버튼 | **비활성화** (opacity 0.4) |
| 인증번호 입력 필드 | **비활성화** |
| 인증 확인 버튼 | **비활성화** (opacity 0.4) |
| 타이머 | `✓ Google` (초록색) |
| `< 이전` 버튼 | **표시** (Step 0 복귀) |

> **참고**: Google 인증 배지("Google 인증 완료 — email")는 제거되었습니다.  
> 이메일 입력 필드에 Google 이메일이 자동으로 채워지므로 별도 표시가 불필요합니다.

### 4-4. 기존 회원 처리 (Google 경로)

`admin.php?action=fan_google_session_set` 응답에서 `already_registered: true` 수신 시:

> **알림 메시지**: 이미 Vibe Studio Fan으로 등록되어 있어요! 신청 항목은 언제든 변경할 수 있습니다. ✨

- UI 비활성화: 동일 (이메일/OTP 필드 비활성화 유지)
- `신청하기` 버튼: **활성화** — 필수 동의 후 항목 업데이트 가능
- **항목 변동 여부와 무관하게 제출 시 항상 DB 업데이트** 허용

## 5. Step 1-B — 이메일 직접 입력 경로

### 5-1. 진입

`이메일로 직접 입력` 클릭 시:
- 내부 상태: `manual`
- Step 2 화면으로 이동 (이메일 입력 활성화)
- `< 이전` 뒤로 가기 버튼 표시

### 5-2. 이메일 인증 발송 (`sendCode`)

| 검증 단계 | 조건 | 메시지 |
|-----------|------|--------|
| 이메일 형식 | `@` 미포함 또는 빈 값 | `올바른 이메일 주소를 입력해주세요.` |
| 쓰로틀링 차단 | 10분 차단 중 | `10분 후 다시 시도하세요.` |
| 쓰로틀링 | 1분 내 3회 초과 | `10분 후 다시 시도하세요.` |
| **기존 회원 이메일** | DB에 이미 등록된 이메일 | `이미 Vibe Studio Fan으로 등록되어 있어요! 신청 항목은 언제든 변경할 수 있습니다. ✨` |
| 메일 발송 실패 | SMTP 오류 | `메일 발송에 실패했습니다.` |
| 서버 오류 | fetch 예외 | `서버 통신 중 오류가 발생했습니다.` |
| 정상 발송 | — | `인증 메일을 발송했습니다.` |

**기존 회원 이메일 시 추가 처리:**
- OTP 입력 필드: **비활성화**
- OTP 확인 버튼: **비활성화** (opacity 0.4)
- 타이머: `✓ 기존` (초록색)
- `신청하기` 버튼: **활성화** — 필수 동의 후 항목 업데이트 가능
- **항목 변동 여부와 무관하게 제출 시 항상 DB 업데이트** 허용
- **서버측 처리**: `send` 액션에서 기존 회원 감지 시 `$_SESSION['auth_verified'] = true` 자동 설정  
  → OTP 입력 없이 `register` 액션의 인증 검증을 통과할 수 있도록 허용

### 5-3. OTP 인증 확인 (`verifyCode`)

| 검증 단계 | 조건 | 메시지 |
|-----------|------|--------|
| 빈 값 | 인증번호 미입력 | `인증번호를 입력해주세요.` |
| 세션 없음 | 인증 정보 없음 | `인증 정보가 없습니다.` |
| 만료 | 10분 경과 | `인증 시간이 만료되었습니다.` |
| 코드 불일치 | 잘못된 코드 | `인증번호가 올바르지 않습니다.` |
| 성공 | — | `인증 성공! 이제 사전예약을 완료해주세요.` |
| 서버 오류 | fetch 예외 | `서버 통신 중 오류가 발생했습니다.` |

**인증 성공 시:**
- 타이머: `✓ 완료` (초록색)
- 타이머 카운트다운: 정지

### 5-4. OTP 타이머

| 항목 | 값 |
|------|----|
| 유효 시간 | 10분 (600초) |
| 표시 형식 | `MM:SS` (빨간색) |
| 만료 표시 | `만료` |
| 만료 메시지 | `인증 시간이 만료되었습니다. 다시 발송해주세요.` |

## 7. Step 3 — 신청 제출 및 완료

### 7-1. 제출 전 검증 (`submitReg`)

| 검증 | 조건 | 메시지 |
|------|------|--------|
| 필수 동의 | `[필수]` 항목 미체크 | `필수 항목에 동의해주세요.` |
| 이메일 인증 | Google / 기존 / 완료 상태가 아닌 경우 | `이메일 주소 인증을 먼저 완료해주세요.` |

**인증 완료 판단 기준 (timer 텍스트)**:

| 텍스트 | 의미 |
|--------|------|
| `✓ Google` | Google OAuth 인증 완료 |
| `✓ 완료` | OTP 이메일 인증 완료 |
| `✓ 기존` | 이미 등록된 이메일 (OTP 불필요) |
| `이미 google_done 상태` | `_fanFlowState === 'google_done'` (검증 생략) |

### 7-2. 신청 성공

- 버튼 텍스트: `신청 완료! 🎉`
- 버튼 배경: 초록 그라디언트 (`#34d399 → #10b981`)
- 알림 메시지: `Vibe Studio Fan 신청이 완료되었습니다!`
- 2.5초 후 패널 자동 닫힘 및 초기화 (Step 0 복귀)
- 완료 이메일 자동 발송 (신규/업데이트 구분)

> **업데이트 정책**: 기존 회원이 동일한 항목 그대로 제출해도 오류 없이 항상 DB 업데이트 허용. 항목 변동 여부는 검증하지 않습니다.

### 7-3. 신청 실패

| 상태 | 메시지 |
|------|--------|
| 서버 오류 응답 | 서버 반환 메시지 표시 |
| fetch 예외 | `서버 통신 중 오류가 발생했습니다.` |

## 9. UI 상태 플래그

### `_fanFlowState`

| 값 | 의미 |
|----|------|
| `idle` | 초기 상태 (Step 0) |
| `google_pending` | Google 팝업 진행 중 |
| `google_done` | Google 인증 완료, Step 2 진입 |
| `manual` | 이메일 직접 입력 모드, Step 2 진입 |

### 알림 영역 타입 (`setNotify` type)

| type | 색상/아이콘 | 사용 시점 |
|------|------------|----------|
| `success` | 초록 / check-circle | 인증 완료, 신청 완료, 기존 회원 안내 |
| `error` | 빨강 / alert-circle | 유효성 오류, 인증 실패, 서버 오류 |
| `info` | 파랑 / loader-2 | 처리 중 (메일 발송 중 등) |
| `''` (기본) | 기본 / info | 안내 메시지 |

## 11. 보안 정책

| 항목 | 구현 방법 |
|------|----------|
| CSRF (Google OAuth) | `fan_oauth_state` 세션 토큰 (16바이트 난수), 콜백에서 검증 |
| postMessage Origin | `event.origin === window.location.origin` 검증 |
| 세션 고정 방지 | Google 콜백 처리 후 `session_regenerate_id(false)` — 구 세션 즉시 삭제 안 함 (AJAX 연속 요청 안정성) |
| 세션 저장 API | `HTTP_ORIGIN` 헤더로 출체 검증 (SITE_URL과 일치해야 함) |
| Google 세션 라이프사이클 | `fan_google_*` 키는 `register` 성공 후에도 유지 — 명시적 로그아웃(`fan_google_session_clear`)시만 삭제 |
| OTP 쓰로틀링 | 1분 내 3회 초과 시 10분 차단 (세션 기반) |
| 이메일 Origin 검증 | `mail_auth.php` 전체 요청에 허용된 Origin만 통과 |
| 관리자 인증 | `$_SESSION['admin_auth'] === true` 세션 확인 |
| 기존 회원 OTP 우회 | `send` 액션에서 기존 이메일 감지 시 `$_SESSION['auth_verified'] = true` 자동 설정 |
